# Provenance / cognition source notes (2026-03-14)

## reAnalyst: Scalable Analysis of Reverse Engineering Activities
- URL: https://arxiv.org/html/2406.04427v1
- Key points captured this run:
  - Tool-agnostic collection of screenshots, keystrokes, active processes/windows, optional annotations.
  - Goal is scalable study of RE practice via semi-automated high-level activity extraction.
  - Frames manual annotation of low-level logs as a central bottleneck.
  - Useful for KB because it turns RE workflow observation into an explicit methodological object.

## SensorRE / Analytic Provenance for Software Reverse Engineers
- AFIT dissertation landing page: https://scholar.afit.edu/etd/3882/
- DTIC PDF metadata/snippet: https://apps.dtic.mil/sti/trecms/pdf/AD1108805.pdf
- Key points captured this run:
  - Reverse engineering is framed as open-ended exploration with multiple possible explanations and context-maintenance burden.
  - SensorRE is presented as the first analytic provenance tool for software reverse engineers.
  - Graph/storyboard style views and automatic capture of sensemaking actions are central design elements.
  - Evidence supports treating provenance and evidence trails as first-class RE workflow support.
