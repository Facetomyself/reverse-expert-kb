## New findings (conservative)
Retained practical ladder:

```text
timer object created/set
  != timer still armed at the moment that matters
  != callback queued truth
  != callback not already cancelled/replaced
  != callback running truth
  != first behavior-changing consumer truth
```

Key retained operator consequences:
- `CreateTimerQueueTimer` creates a timer-queue timer and calls a callback when the timer expires
- periodic timers can queue callbacks even if the previous callback has not finished
- `SetThreadpoolTimer` replaces the previous timer and NULL due time stops new callbacks from being queued, but already queued callbacks can still occur
- cancellation/close paths must be treated carefully because one callback may already be scheduled or past recall
