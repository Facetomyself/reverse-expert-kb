## New findings
Primary durable triage rule added this run:

```text
callback/dispatch crash
  != PAC-shaped callable-pointer failure
  != generic selector/signature mismatch
  != block-lifetime / capture / object-ownership failure
  != later consumer/state consequence problem
```

Concrete practical reinforcements preserved in the note:
- not every arm64e callback crash is actually a PAC problem
- callback landing truth should stay separate from callable-pointer truth
- callable-pointer truth should stay separate from later resume/delivery truth
- delivery truth should stay separate from later consumer/state consequence truth
- copied/reinterpreted/stale callable pointers, analyst-built trampolines, or block invoke-pointer/layout mistakes are narrower candidate seams than broad PAC folklore until proved
