## New findings
The main source-backed takeaway preserved this run:

- continuation resume truth is strong handoff proof, but it is still weaker than the later reducer / coordinator / state owner that changes app behavior
- `@MainActor` isolation is a separate runtime truth boundary worth preserving when the resumed task itself is not the durable consumer
- some compare pairs should be framed as:
  - same callback family
  - same continuation resume or stream delivery
  - different later MainActor-side route/state outcome
- in those cases, reopening generic callback hunting is the wrong move; the better next step is to freeze one MainActor-side consumer and one later effect

After this run, the branch more honestly preserves a five-part operator chain for these cases:
- callback / completion truth
- continuation resume or stream delivery truth
- resumed task-side reducer truth

- later visible effect

That is practical because it helps analysts avoid:
- overreading raw callback or resume visibility as final meaning
- flattening actor isolation and UI-state ownership into generic async drift
- handing off too early from continuation routing into result-to-policy work
