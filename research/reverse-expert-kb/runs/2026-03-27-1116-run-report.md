## Why this branch / seam
Recent runs already kept the anti-stagnation rule alive with real external passes, but this workflow should not drift into endless internal sync.
This run therefore still performed a real explicit multi-source search and targeted a thinner, practical protocol/firmware seam rather than polishing already-dense branches.

The chosen seam is useful because analysts often stop too early at:
- completion bytes are visible
- interrupt/callback fired
- used entry exists
- request looks done

But later replay, emulation, ring-stall diagnosis, and ownership reasoning often need the stronger proof object:
- the representative completion was actually consumed
- the slot/head/CQ position was explicitly advanced or seen
- the slot became durably reclaimable/reusable

Compact shorthand added this run:
- `published != notified != completion-visible != consumed != reclaimed/reusable`
- especially: `completion-visible != slot-returned`

## Practical synthesis added this run
The main KB contribution from this run is a sharper completion-side stop rule for descriptor/CQ workflows:
- publication is not completion
- notification is not completion truth
- completion visibility is not yet consumption/reclaim truth
- slot return/reuse is often the stronger proof boundary later experiments actually need

This is practical because it changes the analyst’s next move.
Instead of broadening queue taxonomy again, the workflow can now freeze one narrower edge such as:
- first `seen` / head advance
- first reclaim index write
- first free/return path
- first slot reuse that depends on earlier consumption
