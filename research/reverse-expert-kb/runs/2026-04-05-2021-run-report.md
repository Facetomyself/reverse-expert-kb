```text

  != current session/webview uses it

  != first interception consumer proved
```

- registration and current ownership are different truths (`registerClass`, `protocolClasses`, `setURLSchemeHandler`)

- shared session, custom session, and WebView configuration should be kept separate before overclaiming request ownership
