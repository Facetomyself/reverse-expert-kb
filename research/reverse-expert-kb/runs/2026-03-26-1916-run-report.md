## New findings preserved
The main practical refinement preserved this run is:
- GUI-framework visibility should not be flattened into ownership truth

The branch should preserve three distinct practical splits:

### 1. Win32 helper subclass identity
A reused helper callback alone is weaker than the truthful proof object.
For helper-based `SetWindowSubclass(...)` cases, keep:
- exact `HWND`
- callback address
- subclass ID
- instance-local reference data

That is more truthful than stopping at “this callback subclasses the control.”

### 2. Qt event-filter fate versus later consumer ownership
`installEventFilter(...)` visibility or `eventFilter(...)` hit visibility is weaker than behavior ownership.
Keep the split:
- filter suppresses / rewrites / retargets event
- or filter returns `false` and later object handler / direct slot / queued slot owns the first durable consequence

That means filter execution is not automatically first-consumer proof.

### 3. Global GUI hook visibility versus later local owner
Application-global or early framework hooks can be strong reduction boundaries while still being weaker than the later truthful consumer.
Examples:
- application-wide Qt filter
- `NSApplication sendEvent:`
- other early shared GUI dispatch hooks

The practical stop rule is:
- if the global hook does not actually change fate, continue until one per-object, per-responder, per-receiver, or per-slot consumer predicts the downstream effect
