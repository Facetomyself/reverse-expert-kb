## Practical synthesis added this run
The main KB contribution from this run is a more truthful stop rule for Windows exception-owned protected-runtime work:
- seeing a dynamic unwind registration API is not the same as proving that the current branch-defining PC is inside that owned region
- current-PC range plausibility is not the same as proving that active-table lookup really resolves that PC into one runtime-owned entry
- a successful lookup hit is still not the same thing as proving the later resumed branch or consequence that matters

Useful branch shorthand:
- `registered != covering this PC != lookup hit != resumed consequence`

That shorthand is practical because it tells the analyst exactly what to freeze next instead of broadening into more unwind theory.
