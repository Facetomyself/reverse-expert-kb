## New findings
The main practical findings preserved this run are:

1. **Win32 subclass truth is more specific than “the window is subclassed”**
   - `SetWindowSubclass` identifies callbacks by callback address + subclass ID
   - reference data is associated with the combination of `HWND`, callback, and subclass ID
   - helper-based subclassing cannot cross threads
   - therefore one shared-looking wrapper is not automatically one truthful owner; the smaller truthful object can be one exact `HWND` + subclass hop + callback/ID pair + instance-local reference data

2. **Qt signal visibility is weaker than delivery-mode truth**
   - `AutoConnection` is resolved based on receiver thread affinity
   - some slot delivery is direct at emit time, some is queued through the event loop
   - therefore “signal found” or “queued slot probably later” is too weak once the branch has already narrowed to first-consumer proof

3. **Cocoa `sendEvent:` is often still only a routing boundary**
   - Apple’s event architecture clearly places `NSApplication` and `sendEvent:` early in dispatch
   - but many event families continue into `NSWindow`, view/responder-chain, or target/action receivers
   - therefore `sendEvent:` should only be treated as the first real consumer when it itself suppresses, rewrites, retargets, or policy-gates the later path
