## Practical synthesis added this run
The main KB contribution from this run is a sharper Windows-native async ownership reminder:
- event observation is one proof object
- IOCP packet dequeue is another
- TP_IO callback delivery is another

Useful branch shorthand:
- **`explicit event signaled != IOCP packet dequeued != TP_IO callback delivered`**

Why that matters operationally:
- seeing an explicit event become signaled should not automatically become “the IOCP worker ran”
- seeing `CreateThreadpoolIo(...)` should not automatically become “this operation reached the callback”
- seeing no callback after immediate success should first trigger a check for `FILE_SKIP_COMPLETION_PORT_ON_SUCCESS` and `CancelThreadpoolIo(...)`, not immediate speculation about dead callbacks or alternate consumers

That is exactly the kind of smaller, case-driven stop rule the native branch should preserve.
