## New findings
The main practical refinement preserved this run is:
- callback-heavy interrupt paths should not flatten everything into one vague story of “the ISR did it”

The branch should preserve four separate proof objects:
1. **arm/publish truth**
2. **IRQ arrival truth**
3. **deferred scheduling truth**
4. **first consumer / reduction truth**

That matters because two different practical subcases recur:

### Subcase A: immediate interrupt-side consequence really is enough
If an IRQ-side path directly performs the decisive status reduction or durable state write, then:
- the IRQ/top-half-adjacent object may already be the first truthful consumer
- the analyst should not invent a later worker story unless stronger evidence requires it

### Subcase B: producer -> deferred handoff -> later policy/state ownership
If the interrupt-side code mainly acknowledges, masks, snapshots status, or schedules later work, then the practical stop rule is:
- freeze one scheduled/deferred carrier first
- then freeze one later consumer reduction such as a poll loop, worker-side state write, reply selector, or wakeup edge
- do not stop at ISR naming or deferred-family inventory alone

A second useful refinement preserved this run:
- NAPI-like and workqueue-like cases often make the ownership split explicit
- the fact that an IRQ scheduled later processing is weaker than proving which later consumer drained the ring/status/completion material and changed durable behavior

## Source-backed synthesis
The useful cross-source synthesis here is:
- generic IRQ and deferred-work material establish that interrupt arrival, bottom-half/softirq/tasklet/workqueue scheduling, and later processing context are distinct execution layers
- workqueue material reinforces that some meaningful behavior only becomes possible in later process-context work rather than in atomic interrupt-side code
- NAPI material is especially useful because it makes the handoff explicit: scheduling and later poll ownership/completion release are not the same proof object

The resulting branch memory is sharper:
- not just “interrupts and deferred work matter”
- but “armed, observed IRQ, scheduled deferred work, and consumed consequence are distinct proof layers; stop at the smallest one that actually predicts behavior”

## Concrete scenario notes or actionable tactics added this run
Preserved tactics:
- keep the explicit shorthand:
  - `armed != observed_irq != scheduled_deferred != consumed`
- if the interrupt-side path already performs the decisive reduction and predicts the symptom, stop at that IRQ-side consequence object first
- if the top-half mainly acks/masks/schedules, prefer the first queue node, poll owner, worker argument package, reply selector, or durable state write over more ISR-family inventory
- do not overread NAPI/workqueue scheduling as proof of actual consumer ownership
- ask what first object survives callback scope and predicts later behavior better than the interrupt label itself
