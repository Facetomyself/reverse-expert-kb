## New findings (conservative)
Retained practical ladder:

```text
same opnum/arguments
  != same binding-handle auth-info truth
  != same authenticated call posture
  != same live context-handle lineage
  != same comparable server-side call
```

Key doc-backed operator consequences retained:
- `RpcBindingSetAuthInfo` sets authentication/authorization information on a binding handle
- unless it is called, RPCs on that binding are not authenticated
- `RpcBindingInqAuthInfo` can show the binding’s effective auth-info posture and can return `RPC_BINDING_HAS_NO_AUTH`
- the runtime may upgrade the effective authentication level above the originally requested level
- context handles are opaque client-side tokens for server-maintained state, so copying visible bytes is weaker than reproducing lineage
