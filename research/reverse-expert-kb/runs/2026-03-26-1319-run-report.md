Useful practical source themes retained:

- explicit CPU/device sync boundaries matter on some targets even when the bytes already look present in memory

## New findings
The main source-backed takeaway preserved this run:
- in descriptor-driven cases, **published != notified != trusted != reclaimed**

Concrete retained examples:
- a driver can prepare descriptors, publish them via an index/owner/freshness move, and only then notify the peer
- a completion slot can look structurally populated while still being weaker than a proved CPU-trust boundary on non-coherent or streaming-DMA paths
- interrupts and doorbells are often useful correlation anchors, but weaker than one proved ownership/freshness/sync transition
- the practical first classification may need to be: coherent shared descriptor memory or streaming/non-coherent DMA-backed visibility?

## Reflections / synthesis
This run materially improves a real protocol/firmware trap that causes bad stopping behavior:

Before this run, the KB already preserved:
- publication-before-consumption logic
- freshness/owner semantics
- reclaim / slot-reuse importance
- non-coherent visibility pitfalls in general

What still needed stronger branch memory:
- a clearer stop rule that **notify/kick/doorbell is not automatically trust transfer**
- a stronger operator vocabulary that distinguishes prepared work, published work, notified work, trustworthy consumer visibility, and reclaimed work
- wider canonical synchronization so this distinction is not trapped in one source note or one leaf page

After this run, the branch more honestly preserves a smaller ladder:
- prepare descriptor/buffer truth
- publish via owner/index/freshness movement
- note any notify/doorbell correlation separately
- prove the consumer-side trust boundary, including explicit sync when the memory class requires it
- prove reclaim / slot reuse as the durable completion-side consequence

That is practical because it helps analysts avoid:
- stopping at easy-to-observe interrupts or doorbells
- misreading RAM-visible completion bytes as enough proof on non-coherent paths
- flattening several different ownership transitions into one vague "queue happened" story
