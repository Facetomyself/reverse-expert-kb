## Practical operator value added
This run improved a real analyst stop rule.

Before this refinement, the iOS branch could still allow a slightly too-flat reading:
- callback truth is good enough, so the visible Swift `async` surface probably owns the consequence
- the continuation resumed, so the visible UI or view-model effect probably belongs to that same moment
- `@MainActor` annotation exists, so the behavior-bearing consumer is already obvious

After this refinement, the branch more honestly supports:
- **callback truth** -> what event family is actually frozen
- **continuation creation/storage truth** -> what bridge was prepared
- **resume/delivery truth** -> what async-visible handoff actually fired
- **first resumed reducer truth** -> where resumed material first becomes app-shaped
- **actor-hop / executor-handoff truth** -> whether the first behavior-bearing consumer actually sits one hop later
- **MainActor-side consumer truth** -> which concrete state mutation / route selection / coordinator handoff predicts the later effect

That changes real case handling:
- visible Swift `async` surfaces are less likely to be overclaimed as final ownership
- resume-time visibility is less likely to be narrated as if it already proved the UI-state consequence
- `@MainActor` annotations are less likely to be mistaken for consumer proof on their own
- compare pairs are less likely to collapse distinct failure shapes like same callback, same resume, different actor-hop, or same hop but different MainActor consumer

This is practical operator value because it tells the analyst what **not** to conclude too early in modern Swift-heavy iOS cases, which is often the real bottleneck.
