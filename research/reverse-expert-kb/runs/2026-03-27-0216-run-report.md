## Practical synthesis added this run
The main KB contribution from this run is a sharper rule for service-oriented protocol reversing:
- seeing a method roster is not the same as proving that the runtime registered or exposed that method family
- seeing a registration or dispatch structure is not yet the same as proving live reachability or consequence
- the workflow should therefore stop at one representative registration/binding/dispatch anchor before broadening into replay or deeper semantics

Useful branch shorthand:
- `described != registered != reachable`
