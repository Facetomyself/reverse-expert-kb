## Practical synthesis added
The new workflow note centers on the reusable ladder:

```text
plist artifact exists
  != load-eligible job under the right domain/ownership constraints
  != loaded/bootstrap state in launchd
  != actual trigger/start in the run that matters
  != relaunched/kept-alive runtime truth
  != later malware-owned effect
```

Key practical reminders preserved in the new leaf:
- LaunchAgent vs LaunchDaemon path placement changes the truthful runtime question (session/login vs system-domain semantics)
- `Disabled` in the plist is not a trustworthy runtime-state oracle
- `RunAtLoad` is one initial-launch condition, not later durable-effect proof
- `KeepAlive` may be unconditional or conditional; the real proof object may be the satisfied condition or relaunch boundary, not the raw key presence
- analyst-driven `launchctl start` is debugging/narrowing truth, not natural durable-trigger proof
