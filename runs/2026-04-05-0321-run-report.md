## New findings (conservative)
Retained practical ladder:

```text
job artifact exists
  != parsed / eligible scheduler object
  != minute/day/timestamp match in the run that matters
  != dispatch-owned command or run-parts selection
  != later malware-owned process/effect
```

Key retained operator consequences:
- user crontabs and system-wide cron files differ materially because system-wide formats include a username field
- `run-parts`-driven execution means a visible file in a periodic directory is weaker than one actually accepted script
- `@reboot` and minute/day matches are scheduler-owned dispatch boundaries, not mere config details
- anacron job presence is weaker than timestamp/delay eligibility and actual delayed execution
- on Debian-style systems, anacron itself may be activated by cron or systemd timer depending on the environment, so upstream activation and job execution should not be flattened together
