## New findings
Primary durable split reinforced this run:

```text
filter installed
  != filter hit
  != filter returned true and owned event fate
  != receiver-side event()/handler truth
  != signal emitted
  != connection-mode truth
  != slot delivered
  != first consequence-bearing consumer
```

Concrete practical reinforcements preserved in the KB:
- Qt event filters see events before the target object, but if filters return `false`, the target object still sees the event
- application-global filters run before object-specific filters and often serve as reduction boundaries rather than automatic first-consumer proof
- signals are usually immediate; queued delivery is the special case that must be proved rather than assumed
- `Qt::AutoConnection` is receiver-affinity truth, not generic async vibes
- queued delivery depends on the receiver thread’s event loop, so `queued != delivered`
- event-filter topology is thread-constrained: the monitoring object must live in the same thread as the monitored object
