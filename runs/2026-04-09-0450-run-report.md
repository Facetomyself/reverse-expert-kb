## New findings
Primary durable split reinforced this run:

```text
authored CustomAction row exists
  != relevant UI/execute sequence row is reachable in this run
  != row is processed into the expected immediate/deferred/rollback/commit mode
  != the payload/context seen by that mode is what you think it is
  != execution consumer actually runs
  != later malware-owned effect truth
```

Concrete practical reinforcements preserved in the note:
- UI-sequence visibility is weaker than current execute/repair/uninstall-path reachability
- deferred / rollback / commit families are script-mode truth, not just “the same custom action later”
- visible immediate property-setting logic is weaker than the later deferred payload that actually survives through `CustomActionData`
- `NoImpersonate` matters for in-script families; immediate custom actions remain impersonated setup truth
- rollback/commit rows are weaker than proving the success/failure path that can actually fire them
