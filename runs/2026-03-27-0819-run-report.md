## Why this matters
This run converts documentation into a more reusable analyst rule:
- seeing `SetNotifyInterface` is not the same as proving durable callback liveness
- seeing `SetNotifyCmdLine` is not the same as proving the command will run
- seeing transferred state or transferred notification is not the same as proving final file availability

That is a practical case-facing improvement, not just wording cleanup.
