## Why this change matters
Before this run, the packed-startup branch already knew that:
- packer recognition is not enough
- a dramatic post-unpack jump is not automatically the best handoff
- TLS/CRT/startup scaffolding can matter

After this run, the branch more clearly preserves a smaller operator rule:
- raw entry, raw post-unpack transfer, and payload-bearing post-startup handoff are separate proof objects
- a region still dominated by TLS callback replay, CRT startup, constructor dispatch, or callback-array repair is startup truth, not yet payload truth
- therefore the analyst should stop only when one downstream import/module/object/consumer anchor stays useful after startup normalization quiets down
