Key retained operator consequences:
- `CreateWindowEx` sends `WM_NCCREATE`, `WM_NCCALCSIZE`, and `WM_CREATE` before returning
- create-time message handling is a real per-instance ownership boundary
- subclass chains (`SetWindowSubclass` / `DefSubclassProc`) can make the class-registered WndProc weaker than the real active per-HWND owner
- subclass installation is weaker than proof of the first consequence-bearing callback in that chain
