## New findings
The main practical finding from this run is that the protocol / firmware branch benefits from a sharper canonical reminder that **arrival truth, callback-path truth, and slot/index truth are adjacent but not identical proof objects**.

More concretely:
1. **Queue arrival is weaker than current-owner proof**
   - completion-queue or callback-queue return does not by itself prove the runtime still trusts the same live request owner

2. **Stable slot/index visibility is weaker than phase/owner proof**
   - wraps and reuse can preserve a visible index while silently changing ownership generation

3. **Broad correlation success is weaker than waiter-liveness proof**
   - reply-path correctness and outer correlation matching are still weaker than proving the currently trusted pending owner, future, or waiter object

4. **Late reply failures often need lifetime reduction, not parser blame**
   - once broad ownership already looks good enough, the smaller truthful question is often retire/reuse/phase realism rather than more parser or field labeling

## Reflections / synthesis
This run reinforced a practical cross-family operator rule:

```text
same visible slot/tag/correlation still appears plausible
  -> arrival path still looks broadly correct
  -> runtime asks a narrower question: does this still belong to the current live owner?
  -> timeout, cancel, reconnect, wrap, phase change, or object replacement says no
  -> stale-drop / ignore / no-wakeup wins despite the outer match
```

That is a useful KB improvement because it sharpens when analysts should stop expanding parser/correlation narration and instead freeze one smaller lifetime contract.
