## New findings
Retained practical branch rule:

```text
service worker registration exists
  != current page is controlled by the worker that matters
  != relevant fetch-event consumer proved
  != cache/network ownership truth
  != later visible outcome truth
```

Concrete practical reinforcements added this run:
- registration, activation, and current control are different truths
- `skipWaiting()` is weaker than current control truth and should be kept separate from `clients.claim()` / active-controller ownership
- `fetch` listener presence is weaker than one relevant `respondWith()` branch owning the visible outcome
- cache API presence is weaker than proving one cache-hit, network-fallback, or stale-while-revalidate branch actually owned the current response
- current cached response truth and later refreshed/update truth can belong to different smaller objects
