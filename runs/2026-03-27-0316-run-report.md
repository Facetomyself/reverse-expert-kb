## Practical synthesis added this run
The main KB contribution from this run is a sharper Qt delivery-truth rule for native GUI reversing:
- a recovered connection edge is not yet the same thing as direct execution
- queued eligibility is not yet the same thing as actual queued delivery
- actual queued delivery is not yet the same thing as the first behavior-changing consumer
- event-filter visibility is not yet filter-owned consequence unless the filter actually changes event fate

Useful branch shorthand preserved this run:
- `connected != direct != queued != delivered != consumed`
- `filter-installed != filter-hit != filter-fate != later-consumer`

This specifically helps in cases where:
- callback-recovery output makes a connection graph look more semantically complete than it really is
- `moveToThread(...)` or receiver affinity changes make analysts overread `AutoConnection`
- queued delivery is assumed even though receiver-loop liveness is the missing proof object
- same-thread explicit `QueuedConnection` gets flattened into direct/immediate execution by habit
- event filters are visible but return `false`, leaving the actual consumer later in object handling or slot delivery
