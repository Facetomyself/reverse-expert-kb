## Practical operator value added
This run adds a real stop rule that analysts can use in queue/callback-heavy reversing cases.

Before this run, the KB already preserved:
- pair design
- alignment truth before causality truth
- watched-object shrink after the first meaningful divergence

But an async-heavy ambiguity remained:
- if both runs enter the same event loop, queue family, or callback registration region, should early queue order / callback timing drift already count as the first real divergence?

After this run, the branch more honestly preserves a narrower split:
- broad async subsystem entry
- delivery-class alignment
- first behavior-bearing delivered callback / dequeued work item / consumer boundary
- watched-object reduction
- later reverse-causality or branch-specific proof

That is practical because it helps analysts:
- avoid overreading cross-thread position order or scheduler churn as explanation
- compare one queue-owned object / callback family / consumer-bearing boundary instead of worshipping raw mixed-thread chronology
- ask smaller TTD/rr queries with better correctness and lower noise
