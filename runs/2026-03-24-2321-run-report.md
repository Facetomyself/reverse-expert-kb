## New findings
The main practical finding from this run is that the malware request-builder -> send-boundary page needed to preserve two thinner but very real proof shapes that are easy to flatten away if the branch thinks only in raw `send` / `recv` terms:

1. **Async wrapper proof shape**
   - for WinHTTP / WinINet / URLMon-style async flows, the truthful local proof boundary may be a callback-confirmed request-context or handle-state transition
   - this can be more trustworthy than merely seeing the initiating send-looking wrapper call

2. **Durable job / transfer proof shape**
   - for BITS-style abuse, the truthful outbound handoff may be durable job creation / resume / callback-registration ownership
   - immediate packet visibility may be later, noisier, or less locally attributable than the job-owned handoff itself

This materially improves the malware branch because it avoids a recurring analyst mistake:
- stopping at “a request builder exists”
- or forcing all send-boundary proof into raw socket or obvious immediate-send vocabulary

## Reflections / synthesis
This run did not discover a brand-new top-level branch; it improved a practical continuation inside an already-valuable malware ladder.

The useful synthesis is:

```text
request family already plausible
  -> wrapper family / object model identified
  -> request or transfer becomes owned by one handle / context / bind / job object
  -> callback-confirmed progression or durable job registration proves commitment better than the initiating call alone
  -> later traffic / download / retry / response behavior becomes trustworthy enough for narrower continuation
```

That is a good fit for the reverse KB because it is:
- practical
- operator-facing
- workflow-centered
- conservative about source claims
- directly useful for breakpoint / hook placement and stop-rule selection

It also keeps malware comms analysis from collapsing into either of two bad extremes:
- vague “networking exists” narration
- premature full protocol storytelling before local ownership is trustworthy
