## Practical operator value added
This run strengthened several live-case rules that are easy to get wrong in GUI-heavy native targets:
- one shared helper callback body is weaker than one recovered per-window subclass tuple
- `DefSubclassProc(...)` visibility proves chain structure, not that the current callback owns the behavior-changing consumer
- `installEventFilter(...)` visibility is weaker than one proved filter-owned fate change
- visible signal discovery is weaker than proving whether delivery is immediate, later queued, or only behaviorally relevant one hop downstream

That is useful in cases where:
- several controls/windows reuse one helper subclass callback
- pseudocode makes subclass wrappers look symmetric when instance-local reference data actually separates ownership
- application-global Qt filters are visible early but the first real consumer still lives later
- analysts overclaim from signal discovery before proving delivery mode or downstream consequence

The practical stop rule preserved by this run is:
- freeze the smallest proof object that actually predicts later behavior
- prefer one exact per-window or per-delivery consumer over broad framework visibility
